# Claude Code 101 Onboarding

Welcome to Claude Code! This interactive guide walks you through the basics hands-on.

## Instructions

Run each step sequentially. After completing each step:
- Show what happened and explain briefly
- Ask "다음 단계로 넘어갈까요?" before proceeding
- Keep explanations concise and friendly (Korean preferred, match user's language)
- **"제가 해드릴게요" 톤**: 사용자에게 명령어를 외우거나 직접 타이핑하라고 하지 않는다. 대신 Claude가 먼저 시연하거나, "제가 해드릴까요?"라고 제안한다.

If the user seems stuck or confused at any step, offer to explain more or skip ahead.

### 용어 설명 원칙

모든 단계에서 기술 용어가 등장하면 **반드시 직관적인 비유나 한 줄 설명을 덧붙인다.** 대상 사용자는 개발자가 아닐 수 있다.

예시:
- **MCP** → "외부 서비스(캘린더, 슬랙 등)를 Claude에 연결해주는 플러그인 시스템"
- **토큰** → "AI가 읽고 쓰는 글자 단위. 대화가 길수록 토큰을 많이 씀 = 사용량 한도 소모"
- **브랜치** → "원본 코드를 건드리지 않고 별도 공간에서 작업하는 것. 문서의 '사본 만들기'와 비슷"
- **PR (Pull Request)** → "내가 수정한 내용을 팀에 검토 요청하는 것. '이거 확인해주세요' 버튼"
- **git** → "파일의 변경 이력을 추적하는 도구. 구글 문서의 '버전 기록'과 비슷하지만 훨씬 강력"
- **스크래핑** → "웹사이트에서 정보를 자동으로 긁어오는 것. 사람이 복사+붙여넣기 하는 걸 기계가 대신"
- **마크다운(md)** → "간단한 기호(#, *, - 등)로 문서를 꾸미는 텍스트 형식. 메모장에서도 쓸 수 있는 가벼운 문서 포맷"
- **컨텍스트(context)** → "AI가 현재 대화에서 기억하고 있는 내용. 대화가 길어지면 컨텍스트도 커짐"
- **세션** → "하나의 대화 단위. Claude Code를 열고 닫을 때까지가 한 세션"

처음 등장하는 용어만 설명하고, 이미 설명한 용어는 반복하지 않는다.

## Step 1: Claude / Cowork / Code 차이점

먼저 Claude 제품군의 차이를 간단히 설명합니다:

| 제품 | 한 줄 설명 | 비유 |
|------|-----------|------|
| **Claude** (claude.ai) | 웹/앱 채팅 AI. 대화로 질문 & 답변 | 카카오톡으로 똑똑한 친구에게 물어보기 |
| **Claude Cowork** (claude.ai/cowork) | 웹에서 문서/코드 실시간 협업 | 구글 문서에서 AI와 함께 편집하기 |
| **Claude Code** (터미널/VS Code) | 내 컴퓨터에서 직접 파일을 읽고, 수정하고, 명령을 실행 | 내 책상에 앉아서 일하는 AI 동료 |

**Explain:** "지금 우리가 사용하고 있는 것이 Claude Code입니다. 채팅 AI와 달리 여러분의 컴퓨터에서 직접 작업합니다. 파일을 열어보고, 수정하고, 프로그램을 실행할 수 있어요."

## Step 2: First Conversation — 파일 읽고 쓰기 체험

Say hello and introduce yourself. Then give the user two options:

"첫 번째 체험을 해볼게요. 두 가지 중 골라주세요:

**A. 소설 체험** — 유명한 소설의 첫 페이지를 **마크다운 파일**(간단한 기호로 꾸미는 텍스트 문서)로 만들어서 저장하고, 다시 읽어드릴게요.
**B. 쿠팡 스크래핑** — 쿠팡에서 원하는 키워드로 검색해서 상품 목록을 자동으로 긁어오고, 파일로 정리해드릴게요. (브라우저 자동화 플러그인 설치가 필요한데, 제가 도와드릴게요!)

어떤 걸 해볼까요?"

### Option A: 소설 체험 (기본)

1. 유명한 소설(어린왕자, 변신, 노인과 바다 등)의 첫 페이지를 한국어로 작성
2. 현재 폴더에 `sample.md` 파일로 저장
3. 저장한 파일을 다시 Read 도구로 읽어서 보여주기
4. 작업 후 `sample.md` 삭제

**Explain:** "방금 파일을 만들고, 다시 읽었습니다. 채팅 AI였다면 텍스트를 보여주기만 하지만, Claude Code는 실제로 여러분 컴퓨터에 파일을 만들고 읽을 수 있습니다."

### Option B: 쿠팡 스크래핑 (브라우저 자동화 체험)

> **스크래핑**이란? 웹사이트에서 정보를 자동으로 긁어오는 것입니다. 사람이 하나하나 복사+붙여넣기 하는 걸 기계가 대신 해주는 거예요.
> 이걸 하려면 **OpenChrome**이라는 브라우저 자동화 플러그인이 필요합니다. **MCP**(Model Context Protocol)라는 시스템으로 Claude Code에 외부 기능을 연결하는 건데, 쉽게 말하면 Claude에 플러그인을 꽂는 거예요.

#### B-1. OpenChrome 설치 확인 및 안내

먼저 OpenChrome MCP가 이미 설치되어 있는지 확인한다:
- 사용 가능한 도구 목록에 `mcp__openchrome__navigate` 등이 있으면 → 설치 완료, B-2로 진행
- 없으면 → 아래 설치 과정을 안내한다

**설치 안내:**

"브라우저 자동화를 하려면 OpenChrome 플러그인을 설치해야 합니다. 한번만 설치하면 됩니다.

두 가지 방법이 있어요:
1. **제가 직접 설치해드리기** — '설치해줘'라고 말씀해주시면 제가 알아서 진행합니다.
2. **직접 명령어 입력하기** — 터미널에 아래 명령어를 입력하면 됩니다:
   ```
   claude mcp add openchrome -- npx @anthropic-ai/openchrome@latest
   ```

어떤 방법이 편하세요?"

사용자가 "설치해줘"를 선택한 경우:
1. Bash 도구로 `claude mcp add openchrome -- npx @anthropic-ai/openchrome@latest` 실행
2. 설치 결과를 보여주고 설명
3. "Claude Code를 `/clear` 후 다시 시작하면 브라우저 도구가 활성화됩니다."

설치가 어려운 경우 → "괜찮습니다, Option A로 해볼게요!" 하고 Option A로 전환

> **Tip:** MCP 플러그인은 한번 설치하면 모든 프로젝트에서 사용할 수 있습니다.

#### B-2. 스크래핑 실행

1. 사용자에게 검색 키워드를 물어본다 (예: "무선 이어폰", "캠핑 의자")
2. OpenChrome MCP로 쿠팡(coupang.com)에서 해당 키워드를 검색
3. 검색 결과 상위 10개 상품의 이름, 가격, 평점을 스크래핑
4. 현재 폴더에 `coupang-search.md` 파일로 표 형태로 정리하여 저장
5. 저장한 파일을 Read 도구로 읽어서 보여주기
6. 작업 후 `coupang-search.md` 삭제

**Explain:** "방금 브라우저를 자동으로 열어서 쿠팡을 검색하고, 결과를 정리해서 파일로 저장했습니다. 터미널 하나로 웹 브라우징까지 자동화할 수 있습니다."

## Step 3: 실행 방법 — 터미널 / VS Code / Desktop App

Claude Code를 실행하는 3가지 방법을 설명합니다:

| 방법 | 누구에게 추천? | 시작 방법 |
|------|--------------|----------|
| **터미널** | 개발자, 파워 유저 | `cd 프로젝트폴더 && claude` |
| **VS Code 확장** | 코드 에디터 사용자 | Extensions에서 "Claude Code" 검색 → Install |
| **Claude Desktop App** | 비개발자, 처음 시작하는 분 | claude.ai에서 앱 다운로드 |

"지금 여러분이 사용하고 있는 건 터미널 방식입니다. VS Code를 쓰신다면 확장 프로그램으로도 똑같이 사용할 수 있어요."

**Tip:** Mac Finder에서 폴더를 **Option(⌥) 누른 채 우클릭**하면 "경로 복사"가 나옵니다. 터미널에서 `cd` + 붙여넣기(⌘V)로 원하는 폴더로 이동할 수 있어요.

## Step 4: 유용한 명령어 체험

각 명령어를 하나씩 **직접 실행해서 보여준다.** 사용자에게 타이핑을 요구하지 말고, Claude가 먼저 시연하거나 "제가 보여드릴게요"라고 한다.

### 4-1. `/usage` — 사용량 확인

"우리 팀은 Claude 구독(Pro 또는 Max)을 쓰고 있어서 별도 요금이 청구되지는 않습니다. 대신 **사용량 한도**가 있어요. `/usage`를 입력해보세요."

> **토큰**이란? AI가 읽고 쓰는 글자 단위입니다. 한글 한 글자가 대략 2~3 토큰 정도예요.

구독 플랜별 차이:

| 플랜 | Opus 사용 | Sonnet 사용 | 한도 초과 시 |
|------|----------|------------|------------|
| **Pro** ($20/월) | 제한적 | 넉넉함 | 몇 시간 대기 후 다시 사용 가능 |
| **Max** ($100/월) | 넉넉함 | 매우 넉넉함 | 대기 시간 짧음 |

"한도를 다 쓰면 잠시 기다렸다가 다시 쓸 수 있습니다. 요금이 추가로 나가는 건 아니에요. `/usage`로 현재 얼마나 썼는지 확인할 수 있습니다."

### 4-2. `/model` — AI 모델 선택

"지금 어떤 AI 모델을 쓰고 있는지 확인할 수 있습니다. `/model`을 입력해보세요."

세 가지 모델을 비유로 설명:

| 모델 | 비유 | 특징 |
|------|------|------|
| **Opus** | 시니어 전문가 | 가장 똑똑하고 신중. 복잡한 작업에 적합. 토큰을 많이 소모 |
| **Sonnet** | 경력 3년차 | 속도와 품질의 균형. 일반적인 작업에 추천 (기본값) |
| **Haiku** | 신입사원 | 가장 빠르고 가벼움. 단순 작업에 적합 |

"용도에 따라 모델을 바꿀 수 있습니다. 간단한 질문에는 Haiku, 중요한 작업에는 Opus를 쓰면 사용량 한도를 더 오래 쓸 수 있어요."

### 4-3. `/compact` — 대화 압축

"대화가 길어지면 토큰이 많이 쌓여서 한도를 빨리 소모합니다. `/compact`를 쓰면 지금까지의 대화를 요약해서 토큰을 절약할 수 있어요. CLAUDE.md(프로젝트 설정 파일) 같은 중요한 정보는 그대로 유지됩니다."

## Step 5: Plan 모드 체험

**Plan 모드**란? Claude에게 "바로 실행하지 말고, 먼저 계획만 세워줘"라고 하는 모드입니다. 실제로 파일을 수정하거나 명령어를 실행하기 전에 "이렇게 할 건데 괜찮을까요?"를 확인할 수 있습니다.

1. "`Shift+Tab`을 누르거나 `/plan`을 입력하면 Plan 모드로 전환됩니다. 한번 해보세요."
2. 사용자가 Plan 모드로 전환하도록 안내
3. Plan 모드에서 간단한 계획을 제안:
   - "이 폴더에 자기소개 문서를 만드는 계획을 세워볼게요"
   - 계획만 보여주고 실행하지 않음
4. "이렇게 Plan 모드에서는 계획만 세우고 실행하지 않습니다. 큰 작업을 시작하기 전에 '이게 맞는 방향인지' 미리 확인하고 싶을 때 유용합니다."
5. "다시 `Shift+Tab`을 누르면 일반 모드로 돌아갑니다."

## Step 6: "이것도 돼?" — 상상력 테스트

Tell the user:

"Claude Code는 까만 터미널 창 하나로 놀라운 것들을 할 수 있습니다:

- 내일 오후 미팅 시간 옮기기 (Google Calendar 연동)
- 경쟁사 웹사이트 스크린샷 찍어서 정리하기 (브라우저 자동화)
- 매일 아침 환율 체크해서 Slack에 보내기 (자동 반복 실행 + Slack 연동)
- 폴더에 있는 계약서 30개 한번에 요약하기 (PDF 일괄 처리)
- 노션 보드 업데이트하기 (Notion 연동)

Step 2에서 체험한 MCP 플러그인 시스템을 통해 이런 외부 서비스들을 연결할 수 있습니다.
궁금한 게 있으면 뭐든 시켜보세요!"

Wait for the user's question and answer it. 만약 사용자가 특정 기능에 관심을 보이면 해당 MCP 플러그인 설치를 도와준다.

## Step 7: 기억 체계 — CLAUDE.md와 Memory

Claude Code의 기억 시스템을 설명합니다:

"Claude Code는 대화를 끝내면(**세션**이 끝나면) 대화 내용을 잊어버립니다. 마치 매일 첫 출근하는 신입사원처럼요. 그래서 중요한 건 '문서'로 남겨둬야 합니다:

| 기억 장치 | 비유 | 역할 |
|-----------|------|------|
| **CLAUDE.md** | 회사 헌법 | 프로젝트 기본 원칙. '우리 팀은 이렇게 일한다' |
| **Rules** | 업무 매뉴얼 | 상황별 자동 적용 규칙. '이런 경우엔 이렇게 해라' |
| **Skills** | 업무 자동화 버튼 | `/명령어`로 실행하는 자동화. 지금 쓰고 있는 `/onboarding`도 스킬! |
| **Auto Memory** | 업무일지 | AI가 대화 중 중요한 정보를 스스로 기록 |

이 기억 장치들은 대화가 끝나도 사라지지 않습니다."

## Step 8: /clear 체험 — 세션 리셋과 기억의 힘

이 단계에서는 `/clear`가 대화를 날리지만, 기억 장치 덕분에 프로젝트 맥락은 유지되는 것을 직접 체험합니다.

**준비:**

1. 먼저 현재 세션에서 간단한 작업을 하나 합니다:
   - 사용자에게 아무 질문이나 하게 하거나, "이 프로젝트에 대해 한 줄로 요약해줘" 같은 질문에 답변
2. 사용자에게 안내:
   - "지금까지 나눈 대화 내용, 기억하고 계시죠? 이제 `/clear`를 한번 쳐보세요. 대화가 전부 사라집니다."

**체험:**

3. 사용자가 `/clear` 실행 후, `/onboarding`을 다시 실행하면 이 스킬이 재시작된다. 이때 아래와 같이 안내:
   - "방금 대화 내용은 전부 날아갔습니다. 저한테 '아까 뭐 했지?' 하고 물어보면 — 모릅니다."
   - "하지만 이 프로젝트가 뭔지, 어떤 규칙을 따라야 하는지는 알고 있습니다. CLAUDE.md와 Rules 파일이 남아있으니까요."

**Explain:** "이게 기억 체계의 핵심입니다. **대화(세션)는 휘발되지만, CLAUDE.md와 Rules는 영구적입니다.** 마치 신입사원이 어제 점심 뭐 먹었는지는 모르지만, 회사 규정은 알고 있는 것과 같습니다."

## Step 9: /init 체험 — 세션 시작의 베스트 프랙티스

"새로운 대화를 시작할 때, Claude에게 '지금 상황 좀 파악해봐'라고 시키는 방법이 있습니다."

**언제 쓰나요?**

1. **매일 작업 시작할 때**: 새로 대화를 열고 `/init`을 치면 Claude가 알아서 프로젝트를 살펴봅니다:
   - 프로젝트 설정 파일(CLAUDE.md), 최근 변경 사항, 진행 중인 작업 등을 확인
   - "이전에 여기까지 했고, 다음에 이걸 하면 좋겠습니다" 같은 제안

2. **처음 보는 프로젝트에 들어갈 때**: `/init`을 치면:
   - 프로젝트 구조를 파악하고 요약
   - CLAUDE.md가 없으면 만들어줄지 제안

**체험:**

1. 사용자에게 `/init`을 실행하게 합니다
2. Claude가 프로젝트를 살펴보고 요약하는 것을 보여줍니다
3. **Explain:** "`/init`은 아침에 출근해서 어제 하던 일을 이어받는 것과 같습니다. 매번 처음부터 설명할 필요 없이, Claude가 스스로 맥락을 파악합니다."
